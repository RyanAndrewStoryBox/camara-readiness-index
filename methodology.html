How we score providers

The CAMARA Readiness Index scores the public-facing API proposition of providers in the CAMARA and Open Gateway space. We assess what a developer or buyer would see if they visited a provider's portal today: the documentation, the sandbox, the onboarding flow, the pricing page, the listed APIs.

We do not score internal engineering, network performance, or capabilities that a provider claims but does not show.

The 7 scoring categories

Documentation quality

20% weight

Does structured, developer-grade documentation exist? Can you find API references, authentication guides, and getting-started instructions without contacting a sales team?

Developer access and onboarding

20% weight

Can a developer sign up and make a first API call? Is there a sandbox? How many steps does it take, and how long before you see a response?

API scope clarity

15% weight

Which APIs are listed? Can you tell which are live, which are sandbox-only, and which are announced but not yet available? Are API names consistent with CAMARA standards?

Commercial clarity

15% weight

Can you see pricing? Is the commercial model explained? Can a developer or procurement team work out what it costs to use the API without requesting a call?

Standards alignment

10% weight

Does the provider reference CAMARA or Open Gateway? Are standard API family names used? Is the provider visibly participating in the standards process?

Production readiness signals

15% weight

Is there evidence of live production use? Case studies, named customers, SLAs, changelogs, or versioned releases?

Geographic and market clarity

5% weight

Is it clear where the APIs work? Does the provider distinguish between live markets and planned rollouts?

How scores work

Each category is scored 0 to 5 using a rubric applied to public evidence. When evidence is missing or too vague, the category is marked "Unknown" and excluded from the weighted average.

The overall score is a weighted average of all scored categories, displayed as X.X / 5 with a readiness label:

High readiness (4.0 to 5.0) Strong, visible, developer-ready proposition
Moderate readiness (3.0 to 3.9) Credible offer with notable gaps
Early-stage (2.0 to 2.9) Emerging, limited public surface
Minimal (1.0 to 1.9) Weak or absent public-facing proposition
Unclear (0.0 to 0.9, or more than 3 unknowns) Not enough evidence to assess

If more than 3 of 7 categories are Unknown, the overall label defaults to "Unclear" regardless of the numeric score.

What we don't do

We don't infer capability from reputation. A well-known operator with no visible documentation scores the same as an unknown startup with no visible documentation. We don't score internal engineering or network quality. We don't accept payment to influence scores. We don't present opinion as evidence.

How we keep scores current

Every assessment carries a date. We refresh scores when a provider's public surface changes. Each provider profile shows when it was last assessed, so you can judge for yourself how current the data is.
